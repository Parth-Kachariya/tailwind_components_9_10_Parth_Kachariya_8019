Pricing Section UI
Project Task Documentation: Pricing Section UI
Task Title
Design & Develop a Pricing Section (Responsive UI)

Link: https://www.creative-tim.com/twcomponents/component/popular-pricing-section/landing



Duration
⏱ 2 Hours

Objective
The objective of this task is to help students practice HTML, tailwind (Flexbox/Grid), and basic UI design concepts by creating a modern pricing section similar to real-world SaaS websites.

By completing this task, students will:

Understand layout structuring
Improve tailwind styling skills
Learn responsive design
Practice clean and reusable code


Task Description
Students are required to design a Pricing Section UI consisting of four pricing cards:

Intro
Base
Popular (Highlighted)
Enterprise
Each pricing card should display:

Plan name
Monthly price
Short description
Feature list with check icons
"Choose Plan" button
One plan (Popular) must be visually highlighted.



UI Requirements
Pricing Section Header
Title: Simple, transparent pricing
Subtitle: No contracts. No surprise fees.
Toggle buttons (UI only): Monthly / Yearly


Pricing Card Structure
Each card must include:

Plan Name (Intro / Base / Popular / Enterprise)
Price (e.g. $19 / Month)
Short description text
Feature list:
All limited links
Own analytics platform
Chat support
Optimize hashtags
Unlimited users
Action Button: Choose Plan


Design Guidelines
Layout
Use Flexbox or tailwind css Grid
Cards aligned horizontally on desktop
Stack cards vertically on mobile
Highlighted Plan (Popular)
Dark background or different color
Slightly larger card size
Label: Popular
Buttons
Rounded corners
Hover effect (color change or scale)




Technical Requirements
Mandatory Technologies
HTML5
tailwind CSS (for styling and layout)
Optional (Bonus)
Font Awesome or icons
Media Queries
❌ JavaScript is not mandatory (Monthly/Yearly toggle can be UI only)
